# Authority Model

Aphelion separates conversational presentation from authority.

## Face And Governor

The face is the persona you talk to. The governor is the authority layer that decides what can be committed to and records evidence. The face cannot grant itself permissions.

## Consent Path

Permissions travel through typed records:

```text
request -> classify -> review -> provision -> attest -> grant -> expose -> observe -> renew/revoke
```

This matters for tools, external accounts, remote children, and any capability crossing a trust boundary.

## Approval Windows

Temporary approval windows are scoped and time bounded. They open an automation gate and a matching grant together. They do not change the standing authority model.

## Principle

Text is presentation. Authority lives in records that can be inspected, expired, renewed, or revoked.
