# First Requests

Start with requests that make Aphelion describe its own state.

```text
Summarize your current setup and tell me what you can access.
```

```text
Run a health check and explain the result in plain language.
```

```text
Inspect the current working directory and tell me what project this is.
```

## File Inspection

```text
Read the README and give me the install steps as a checklist.
```

Aphelion should ask before actions that cross the configured authority boundary.

## Local Maintenance

```text
Check the Aphelion service logs for recent warnings.
```

```text
Verify the deployment and tell me whether it is safe to use.
```

## Repository Work

```text
Create a branch for a documentation fix, update the install guide, run the relevant checks, and open a draft PR.
```

For repository work, keep requests specific. Include the target repo, branch, and expected output when possible.
